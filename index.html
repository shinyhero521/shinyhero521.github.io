什么是上行短信和下行短信

发表于 2023-04-14 分类于 chat

答案

上行短信和下行短信是指短信的发送和接收的方向。上行短信是指用户发送给电信服务提供商或者SP（服务提供商）的短信，一般需要付费。下行短信是指电信服务提供商或者SP发送给用户的短信，一般免费¹⁴。例如，你给10086发了一条查询话费的短信，这就是上行短信；10086回复你话费余额的短信，这就是下行短信。

如何区分

区分上下行的原因可能有多种，根据不同的领域和场景，可能有不同的解释。比如，在铁路中，区分上下行是为了方便管理和调度列车，一般以北京为中心，靠近北京方向为上行，远离北京方向为下行¹。在道路中，区分上下行是为了确定桩号的大小和方向，一般桩号从小到大的一侧是上行²。在通信中，区分上下行是为了提高数据传输的效率和降低时延，一般可以采用上下行解耦技术，在不同的载波或频段进行数据发送和接收³。

one more thing

下行速率和下行包是网络通信中的概念。下行速率是指基站向移动终端发送信息时的传输速率，比如手机或笔记本等无线终端从基站或者网络下载数据的速率³。下行包是指基站向移动终端发送的数据包，每个数据包都有一定的大小和序号。下行速率和下行包之间有一定的关系，一般来说，下行速率越高，下行包的数量和大小也越大。

utf8mb4_unicode_ci 和utf8mb4_general_ci的区别是什么

utf8mb4_unicode_ci 和 utf8mb4_general_ci 是两种常见的 MySQL 字符集和排序规则。它们的主要区别在于它们如何比较和排序字符，特别是对于一些语言和字符集，如 Unicode。

具体来说，utf8mb4_unicode_ci 使用 Unicode 官方的排序规则进行字符比较和排序，因此更准确地反映了各种语言和字符集的规则。例如，它会将不同形式的相同字符视为相同，如大小写不同的字母和不同的变音符号。

而 utf8mb4_general_ci 则使用一种更简单的排序规则，它不考虑字符的语言或文化背景，仅仅比较字符的基本值。这种排序方式不会考虑一些特殊字符，如 Unicode 表情符号，因此在某些情况下可能会产生错误的排序结果。

综上所述，如果您需要对不同语言和字符集进行排序和比较，建议使用 utf8mb4_unicode_ci，因为它提供了更准确和可靠的排序方式。而如果您处理的是比较基本的字符集，可以使用 utf8mb4_general_ci 以提高性能。

其中ci是什么意思

在MySQL中，ci是指不区分大小写（case-insensitive），意味着在排序或比较时，字符的大小写不会影响结果。因此，utf8mb4_unicode_ci 和 utf8mb4_general_ci 的区别在于它们如何比较和排序字符，以及如何处理大小写。ci后缀通常用于排序规则的名称中，用于表示不区分大小写。相对的，MySQL还提供了带有cs后缀的排序规则，表示区分大小写（case-sensitive）。

case-insensitive是什么意思

case-insensitive 是一个术语，指的是在比较字符时，不考虑字符的大小写。例如，当使用 case-insensitive 方式比较字符串 “hello” 和 “HELLO” 时，它们会被视为相同，因为这种比较方式不会考虑字符的大小写。

相对的，case-sensitive 比较则会考虑字符的大小写，因此 “hello” 和 “HELLO” 在 case-sensitive 比较中被视为不同的字符串。

在计算机编程中，case-insensitive 通常用于文本搜索和排序等场景，以提高用户体验和准确性。例如，当用户在搜索引擎或电子邮件中输入关键词时，常常会使用 case-insensitive 比较来匹配查询和搜索结果，以确保无论用户输入的是大写还是小写字符，都能获得正确的结果。

总体报告参数：

  • Samples：测试中发送的请求总数。
  • Average：所有请求的平均响应时间。
  • Median：所有请求的中位数响应时间。
  • 90% Line：所有请求的第90个百分位数响应时间。
  • 95% Line：所有请求的第95个百分位数响应时间。
  • 99% Line：所有请求的第99个百分位数响应时间。
  • Min：所有请求的最小响应时间。
  • Max：所有请求的最大响应时间。
  • Error%：所有请求中出错的百分比。
  • Throughput：每秒钟发送的请求数量。
  • Received KB/sec：每秒钟接收的数据量。
  • Sent KB/sec：每秒钟发送的数据量。

一些常见的配置参数

  1. 线程数：指同时模拟的用户或请求的数量。线程数的设置应根据应用程序的负载和服务器的配置进行调整。
  2. Ramp-up Period（ramp-up时间）：指线程组中启动线程的时间间隔。ramp-up时间的设置可以模拟用户逐渐增加的情况，以免服务器突然承受过大的负载。
  3. 循环次数：指线程组中每个线程要执行的请求或操作的次数。循环次数的设置可以模拟多个请求或操作的情况，以评估系统的性能表现。
  4. 目标服务器或应用程序的URL地址：指被测试的服务器或应用程序的URL地址。测试人员应该确保这个地址是正确的，并能够正确地连接到被测试的服务器或应用程序。
  5. HTTP请求方法：指在测试中使用的HTTP请求方法，如GET、POST、PUT、DELETE等。测试人员应该根据被测试的应用程序选择适当的请求方法。
  6. 请求参数：指在请求中包含的参数，如用户名、密码、查询条件等。测试人员应该根据测试需求和被测试的应用程序，正确地设置请求参数。
  7. 断言：指在测试过程中验证响应的内容和结果是否正确。测试人员可以使用断言来验证响应的状态码、响应时间、返回数据等内容。

其中ramp-up period是什么

ramp-up period（逐渐增加的时间段）指的是在JMeter测试计划运行期间，测试线程从初始状态开始逐渐增加到目标并发数的时间段。换句话说，ramp-up period是指从启动测试到达到最大并发数所需的时间。

在JMeter中，ramp-up period可以通过设置线程组的属性来定义。例如，如果你希望在1分钟内将并发用户数从0增加到100，你可以将线程组的“Number of Threads”属性设置为100，并将“Ramp-up period”属性设置为60秒。

通过逐渐增加测试负载的方式，可以更好地模拟实际使用情况，并且有助于避免过度负载服务器。如果一下子将所有测试线程全部启动，可能会对服务器造成过度压力，从而导致性能测试结果不准确。

需要注意的是，ramp-up period设置的时间应该足够长，以确保所有线程都有足够的时间启动和运行。如果ramp-up period设置得太短，可能会导致某些线程在测试开始时失败。

其中average的单位是

在JMeter的测试报告中，响应时间的平均值（average）的单位是毫秒（ms）。这个值表示所有请求的平均响应时间，是所有请求响应时间之和除以请求总数的结果。

例如，如果一次测试中发送了100个请求，它们的响应时间分别为10ms、20ms、30ms……900ms，那么这100个请求的平均响应时间就是(10+20+30+…+900)/100=455ms。

平均响应时间是性能测试中常用的一个指标，它可以用来评估系统的响应速度。较短的平均响应时间通常意味着系统的性能更好，而较长的平均响应时间可能表明系统存在性能问题或负载过高。

需要注意的是，平均响应时间只是性能测试报告中的一个指标，不能单独用来评估系统性能。在性能测试中，还需要结合其他指标来综合评估系统的性能。

其中number of threads是什么

在JMeter中，number of threads（线程数）指的是同时模拟的用户或请求的数量。线程是JMeter中模拟并发用户或请求的基本单位，它们可以在测试计划中运行多个线程以模拟多个并发用户或请求。

例如，如果你的应用程序需要支持100个并发用户，你可以在JMeter中创建一个线程组，并将线程组的“Number of Threads”属性设置为100。当测试计划开始运行时，JMeter会创建100个线程，并同时模拟100个并发用户向服务器发送请求。

需要注意的是，线程数不应该过高或过低。如果线程数太低，可能无法充分测试应用程序的性能；如果线程数太高，可能会对服务器造成过度压力，从而导致测试结果不准确。一般来说，线程数应该根据应用程序的负载和服务器的配置进行调整。

在性能测试中，线程数是一个重要的指标之一。通过调整线程数，可以模拟不同的并发负载，评估应用程序在不同负载下的性能表现。同时，线程数还可以用来确定测试计划的总并发量。

qps是什么 一般是多少

QPS（Queries Per Second）是一个用于衡量系统性能的指标，表示每秒钟处理的请求数量。在计算机领域，QPS通常用于衡量服务器、数据库、Web应用程序等系统的吞吐量和性能。

QPS的计算方法是将一段时间内处理的请求数量除以该时间段的秒数，得到的结果即为每秒处理的请求数量。例如，如果一个系统在1秒钟内处理了100个请求，那么它的QPS为100。

QPS的大小取决于系统的负载、配置和性能等因素。一般来说，QPS越高表示系统性能越好，但是过高的QPS可能会对系统造成过度压力，导致系统性能下降或系统崩溃。因此，在设计和评估系统性能时，需要综合考虑QPS和其他指标，以确定系统的最佳性能表现。

具体而言，不同的系统和应用程序对QPS的要求也不同。例如，一些轻量级Web应用程序的QPS可能只有几百或几千，而高性能的数据库或消息队列可能需要支持每秒数十万或数百万的QPS。因此，在确定QPS时，需要考虑具体的应用场景和需求。

Loop Count

循环次数的英文是 “Loop Count”。在性能测试工具（如JMeter）中，循环次数通常可以在线程组中设置。

循环次数是指在性能测试中，一个线程要执行多少次特定的操作或请求。循环次数的设置非常重要，因为它会直接影响到测试的结果和准确性。

循环次数的设置应该根据具体的测试需求和被测试的应用程序来确定。一般来说，如果被测试的应用程序需要进行多个请求或操作才能完成一个完整的业务流程，那么循环次数就应该设置为相应的值。例如，如果需要在应用程序中完成登录、搜索、添加、编辑等多个操作，那么循环次数就应该设置为相应的次数，以保证测试能够覆盖所有的操作流程。

另外，循环次数还应考虑到应用程序的负载和性能。如果应用程序本身性能较差或者服务器配置不高，那么循环次数就应该设置为相对较少的次数，以避免对系统造成过度压力。

需要注意的是，循环次数的设置应该与线程数和ramp-up时间相互协调，以保证测试的准确性和可靠性。一般来说，线程数和循环次数越多，测试的负载就越大，测试结果也就越真实。但是如果线程数和循环次数设置过高，就可能对被测试的应用程序或服务器造成过度压力，导致测试结果不准确或系统崩溃。因此，在设置循环次数时需要根据具体情况进行综合考虑。

infinite

“Infinite” 是一个英文单词，意思是无限的、无穷的，也可以表示没有限制、没有终止点等。在性能测试中，”Infinite” 通常被用来指代循环次数设置为无限循环，即执行无限次特定的操作或请求。在JMeter等性能测试工具中，循环次数可以设置为 “Infinite”，表示循环次数没有限制。但是在实际测试中，建议不要将循环次数设置为无限循环，以免对被测试的应用程序或服务器造成过度压力。

jmeter启动的jvm参数如何调大

JMeter启动时会默认使用JVM的一组参数，这些参数会影响JMeter的性能和稳定性。如果需要提高JMeter的性能，可以尝试通过调整JVM的参数来实现。

下面是一些常用的JVM参数，可以在启动JMeter时加入：

  1. -Xms: 设置JVM的初始堆大小。可以使用-Xms参数来指定，例如：-Xms2048m 表示将初始堆大小设置为2GB。
  2. -Xmx: 设置JVM的最大堆大小。可以使用-Xmx参数来指定，例如：-Xmx4096m 表示将最大堆大小设置为4GB。
  3. -XX:MaxPermSize: 设置JVM的最大永久代大小。可以使用-XX:MaxPermSize参数来指定，例如：-XX:MaxPermSize=512m 表示将最大永久代大小设置为512MB。
  4. -XX:MaxMetaspaceSize: 设置JVM的最大元空间大小。可以使用-XX:MaxMetaspaceSize参数来指定，例如：-XX:MaxMetaspaceSize=512m 表示将最大元空间大小设置为512MB。
  5. -XX:+UseG1GC: 使用G1垃圾收集器。可以使用-XX:+UseG1GC参数来启用G1垃圾收集器。
  6. -XX:+HeapDumpOnOutOfMemoryError: 当JVM发生内存溢出错误时，自动生成一个堆转储文件。可以使用-XX:+HeapDumpOnOutOfMemoryError参数来启用。

在Windows系统中，可以通过修改jmeter.bat文件来修改JVM参数。在jmeter.bat文件中，可以找到以下两行代码：

set HEAP=-Xms512m -Xmx512m set NEW=-XX:NewSize=128m -XX:MaxNewSize=128m

可以根据实际情况修改这些参数的值，以提高JMeter的性能。在Linux系统中，可以通过设置环境变量来修改JVM参数。例如，在Bash Shell中可以使用以下命令来设置JVM参数：

export HEAP=”-Xms2048m -Xmx4096m” export NEW=”-XX:NewSize=512m -XX:MaxNewSize=512m”

这些命令将JVM的初始堆大小设置为2GB，最大堆大小设置为4GB，新生代大小设置为512MB。需要注意的是，修改JVM参数可能会影响到JMeter的稳定性和性能，因此需要谨慎调整。

chmod的那些事

发表于 2023-02-25 分类于 chat


在 chmod +x 命令中，x 表示可执行权限。文件的可执行权限允许用户执行（运行）该文件。在Linux和Unix系统中，每个文件都有一个与之相关联的权限集，指定了哪些用户可以对该文件进行哪些操作。文件权限通常由三组权限标志组成，分别是文件所有者权限、文件所属组权限和其他用户权限，它们分别用三个字符 rwx 表示：

  • r (read)：读权限，允许用户查看文件内容。
  • w (write)：写权限，允许用户修改文件内容。
  • x (execute)：可执行权限，允许用户运行该文件。

因此，通过在文件的权限集中添加 x 标志，你可以为该文件添加可执行权限，从而允许用户运行该文件。

+号是什么意思

在 chmod 命令中，+ 符号表示添加权限。使用 chmod +x filename 命令，表示给文件添加可执行权限。类似的，如果你想要添加读权限和写权限，可以使用 chmod +rw filename 命令。

如果不使用 + 符号，而是使用 - 符号，那么就表示移除相应的权限。例如，chmod -x filename 命令会移除文件的可执行权限，使得该文件不再可以被执行。

除了 + 和 - 符号外，还可以使用 = 符号，来显式地设置文件的权限。例如，chmod u=rw,g=r,o=r filename 命令表示设置文件的权限，使得文件所有者拥有读写权限，文件所属组和其他用户拥有读权限。

755 777 等怎么对应的rwx

在Linux和Unix系统中，文件的权限由三组权限标记组成，分别是文件所有者权限、文件所属组权限和其他用户权限，每组权限由三个字符表示，分别表示可读、可写、可执行权限，顺序为 rwx，其中：

  • r 表示可读权限，允许用户读取文件的内容。
  • w 表示可写权限，允许用户修改文件的内容。
  • x 表示可执行权限，允许用户执行（运行）该文件。

在Linux和Unix系统中，通常使用数字来表示不同的权限设置，每个数字对应一种权限组合。其中，最常见的数字权限设置包括：

  • 7：对应权限组合 rwx，表示所有用户都拥有读、写和执行权限。
  • 6：对应权限组合 rw-，表示文件所有者拥有读写权限，但其他用户没有任何权限。
  • 5：对应权限组合 r-x，表示文件所有者拥有读权限和执行权限，但没有写权限，其他用户只有读和执行权限。
  • 4：对应权限组合 r--，表示文件所有者拥有读权限，但没有写和执行权限，其他用户没有任何权限。
  • 0：对应权限组合 ---，表示所有用户都没有任何权限。

因此，755 权限表示文件所有者有读、写和执行权限，文件所属组和其他用户都只有读和执行权限；而 777 权限表示所有用户都拥有读、写和执行权限。

是怎么分出来的7 6 5

在 Linux 和 Unix 系统中，文件的权限用一个三位的八进制数来表示。每一位都代表一组权限标记，从左到右依次为文件所有者权限、文件所属组权限和其他用户权限。每个权限标记位上的值可以是 0、1、2、4、5、6 或 7。

  • 0 表示没有权限
  • 1 表示执行权限
  • 2 表示写权限
  • 3 表示写和执行权限
  • 4 表示读权限
  • 5 表示读和执行权限
  • 6 表示读和写权限
  • 7 表示读、写和执行权限

因此，为了得到一个三位的八进制数，需要将每一组权限标记的值相加，例如：

  • rwx 权限（读、写和执行权限）的值为 4 + 2 + 1 = 7。
  • rw- 权限（读和写权限）的值为 4 + 2 + 0 = 6。
  • r-x 权限（读和执行权限）的值为 4 + 0 + 1 = 5。

因此，当你想要为一个文件设置特定的权限时，需要将每个权限标记的值相加，得到一个三位的八进制数，然后使用 chmod 命令设置文件的权限。例如，使用 chmod 755 filename 命令，将文件的权限设置为 rwxr-xr-x。

Nginx upstream配置说明负载均衡

发表于 2022-01-17 更新于 2023-02-06


nginx 一定不要放在中文目录下 否则起不来

假设Nginx只能代理一台服务器的话，那它也不可能像今天这么火。Nginx能够配置代理多台服务器。当一台服务器宕机之后。仍能保持系统可用。详细配置步骤例如以下：

  1. 在http节点下，加入upstream节点。
  1. 将server节点下的location节点中的proxy_pass配置为：http:// + upstream名称，即
  1. 如今负载均衡初步完毕了。upstream依照轮询（默认）方式进行负载，每一个请求按时间顺序逐一分配到不同的后端服务器。假设后端服务器down掉。能自己主动剔除。尽管这样的方式简便、成本低廉。但缺点是：可靠性低和负载分配不均衡。

适用于图片服务器集群和纯静态页面服务器集群。 除此之外，upstream还有其他的分配策略，分别例如以下：weight（权重）指定轮询几率，weight和訪问比率成正比，用于后端服务器性能不均的情况。例如以下所看到的。10.0.0.88的訪问比率要比10.0.0.77的訪问比率高一倍。

ip_hash（访问ip）， 每一个请求按訪问ip的hash结果分配。这样每一个訪客固定訪问一个后端服务器，能够解决session的问题。

ip_hash还能和weight共用，详见最下面。但是在nginx版本1.3.1之前，不能在ip_hash中使用权重（weight）。

fair（第三方），按后端服务器的响应时间来分配请求。响应时间短的优先分配。与weight分配策略相似。

所有方式

轮询 方式
weight 权重方式
ip_hash 依据ip分配方式
least_conn 最少连接方式
fair(第三方) 相应时间方式
url_hash(第三方) 依据url分配方式

upstream还能够为每一个设备设置状态值，这些状态值的含义分别例如以下：

  1. down 表示单前的server临时不參与负载.
  2. weight 默觉得1.weight越大，负载的权重就越大。
  3. max_fails ：同意请求失败的次数默觉得1.当超过最大次数时，返回proxy_next_upstream 模块定义的错误.
  4. fail_timeout : max_fails次失败后。暂停的时间。

backup： 其他全部的非backup机器down或者忙的时候，请求backup机器。所以这台机器压力会最轻。

java只有值传递

发表于 2021-05-11 更新于 2023-02-06 分类于 Java原理

结论

Java中只有值传递

  1. 基本数据类型参数传值：就是你cv我
  2. 引用数据类型参数传值：你cv了我的”对象的地址”：

    具体点就是你复制了一个新的栈，但指针指向相同的堆内容，只要你不拆我家，那么咱俩都不变；你若改了，那我也改了。有点像引用传递，但是不是(咱俩是两个栈)只是恰好这种机制用了一个堆内存。

定义

  • 值传递（pass by value）是指在调用函数时将实际参数复制一份传递到函数中，这样在函数中如果对参数进行修改，将不会影响到实际参数。

  • 引用传递（pass by reference）是指在调用函数时将实际参数的地址直接传递到函数中，那么在函数中对参数所进行的修改，将影响到实际参数。

  • 简单粗暴

    值传递： 就是我不变，你cv了我，你和我同值，你模仿我

    引用传递：就是把我给你了

  • 理解

你有一把钥匙，当你的朋友想要去你家的时候，如果你直接把你的钥匙给他了，这就是引用传递。这种情况下，如果他对这把钥匙做了什么事情，比如他在钥匙上刻下了自己名字，那么这把钥匙还给你的时候，你自己的钥匙上也会多出他刻的名字。

你有一把钥匙，当你的朋友想要去你家的时候，你复刻了一把新钥匙给他，自己的还在自己手里，这就是值传递。这种情况下，他对这把钥匙做什么都不会影响你手里的这把钥匙。

但是在引用数据类型传值时：你的朋友拿着你给他的钥匙，进到你的家里，把你家的电视砸了。那你说你会不会受到影响？而我们在pass方法中，改变user对象的name属性的值的时候，不就是在“砸电视”么。你改变的不是那把钥匙，而是钥匙打开的房子。
